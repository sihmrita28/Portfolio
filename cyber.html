Cyber Security

Intern Project

Home/ Cyber Security

Objective

The objective of this project was to explore fundamental concepts in cybersecurity by implementing practical encryption and security-related tools using Python. The project involved building multiple tools such as a Caesar Cipher, image encryption using pixel manipulation, a password strength checker, a basic keylogger, and a network packet analyzer. Each task focused on strengthening understanding of data protection, secure communication, and ethical hacking principles.

Methodology

The tools were developed using Python, leveraging its extensive standard library and simplicity for prototyping security concepts. For the Caesar Cipher, a shift-based encryption logic was implemented that allowed users to input a message and a shift key for both encryption and decryption. In the pixel manipulation task, image processing was used to encrypt and decrypt images by altering pixel values mathematically. The password strength checker assessed input passwords based on length, character types, and complexity criteria, returning real-time feedback. A basic keylogger was built to log keystrokes, saving them to a fileâ€”emphasizing the importance of ethical implementation. Finally, a network packet analyzer was created using libraries such as scapy to capture and inspect real-time network traffic, showcasing the significance of packet-level data visibility in cybersecurity.

Tool Features and Highlights:
Each tool included a basic user interface through terminal prompts or file I/O. The Caesar Cipher and password checker provided real-time encryption feedback and usability. The image encryption tool processed visual data using simple loops and image libraries. The packet analyzer displayed IPs, protocols, and payload data to simulate intrusion detection scenarios. Care was taken to ensure that all tools were used strictly for educational and ethical purposes, especially for sensitive tools like the keylogger and packet sniffer.

User Interaction and Experience:
Users interacted with these tools primarily through command-line input. Each program was built to be beginner-friendly, guiding the user through each step of encryption, analysis, or feedback. The password checker returned feedback on strength, while the image tool output encrypted versions of the original input. For tools like the keylogger and packet analyzer, results were saved to external files for analysis, and user permissions and ethical warnings were clearly defined.
